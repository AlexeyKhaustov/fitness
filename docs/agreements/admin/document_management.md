# Инструкция для администраторов: управление документами

## Где находятся документы?

В админ-панели Django перейдите в раздел **Legal** (или **Юридические документы**). Там вы увидите три модели:

- **Документы** — список типов документов (Политика, Соглашение, Оферта).
- **Версии документов** — все созданные версии с возможностью активации.
- **Согласия пользователей** — журнал, кто и когда принимал те или иные версии.

## Как создать новую версию документа?

1. В списке **Документы** найдите нужный тип (например, «Политика конфиденциальности»).
2. Нажмите кнопку **«➕ Создать новую версию»** в строке этого документа.
3. Вы будете перенаправлены на страницу создания новой версии. Поле `version_number` заполнится автоматически (следующий номер).
4. В поле `text` введите новый текст документа. Можно скопировать старый из предыдущей версии и отредактировать.
5. Поле `is_active` оставьте **НЕ отмеченным** (активировать нужно будет отдельно).
6. Сохраните версию.

## Как активировать версию?

1. Перейдите в раздел **Версии документов**.
2. Найдите нужную версию (удобно фильтровать по документу).
3. Если версия ещё не активна, рядом с ней будет кнопка **«Сделать активной»**.
4. Нажмите её. Все остальные версии этого документа автоматически деактивируются, а текущая версия документа обновится.

После активации пользователи при следующем входе увидят страницу принятия новых условий.

## Как посмотреть, кто дал согласие?

В разделе **Согласия пользователей** вы увидите таблицу с фильтрами по документу и дате. Можно найти конкретного пользователя по email или имени.

## Важные замечания

- Не редактируйте текст напрямую в активной версии — всегда создавайте новую. Иначе пользователи не увидят изменения и не дадут нового согласия.
- Если нужно срочно откатить изменения, активируйте предыдущую версию. При этом пользователи, уже принявшие новую, останутся с ней, но новые пользователи будут видеть старую версию. Это допустимо в исключительных случаях.
- При создании новой версии всегда проверяйте, что текст корректен (орфография, ссылки). Рекомендуется копировать из предыдущей версии и вносить правки.
